# Portfolio Audit — Shadow Oak Capitals (S1–P)

> Final report scoring repositories included in this sprint/workspace.

## Scored repositories

- **SovereignFinance** (personal finance tracker)
- **NifftyVault** (quantitative analytics)
- **OakLedger** (trade & behavioural analytics)
- **Beginner Projects**: List Analyzer, Calculator, Emoji Converter, Rock Paper Scissors

## Scoring rubric (1–10)

- **Architecture**: modularity, layering, clean interfaces
- **Code quality**: typing, linting readiness, readability
- **Docs**: clarity, architecture overview, usage examples
- **Tests**: coverage, meaningful assertions
- **Maintainability**: refactorability, dependency hygiene
- **Dev experience**: CLI usability, config consistency, CI
- **Portfolio value**: signal to recruiters/quant teams
- **Production readiness**: CI quality, error handling, release basics

## Ranking (best first)

1. **NifftyVault** — strongest quant-statistics signal, good structure and documentation.
2. **SovereignFinance** — solid engineering core and dev workflow; docs moderate.
3. **Beginner Projects** — good for interviews (fundamentals), but lower “portfolio value” for quant.
4. **OakLedger** — earliest-stage quality; needs deeper refactor to reach the standard of the others.

## Recommended portfolio order

### For recruiters (general SWE)
1. SovereignFinance
2. NifftyVault
3. Beginner Projects (as supporting evidence)
4. OakLedger (only if you can explain the missing engineering rigor)

### For quant / finance teams
1. NifftyVault
2. OakLedger
3. SovereignFinance
4. Beginner Projects

### For software engineering interviews
1. NifftyVault
2. SovereignFinance
3. Beginner Projects
4. OakLedger

- Score totals are intentionally conservative where CI/tests/lint weren’t verified in this environment.
